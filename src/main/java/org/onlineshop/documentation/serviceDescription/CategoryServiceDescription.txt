Описание сервиса CategoryService
----------------------------------

1. Назначение класса:
   Класс CategoryService отвечает за бизнес-логику, связанную с управлением категориями товаров в онлайн-магазине.
   Он реализует интерфейс CategoryServiceInterface и выполняет операции добавления, обновления, удаления и получения категорий.

2. Основные зависимости:
   - CategoryRepository — доступ к базе данных (работа с сущностью Category).
   - CategoryConverter — преобразование между сущностью Category и DTO-классами CategoryRequestDto / CategoryResponseDto.

3. Основные методы:

   • addCategory(CategoryRequestDto categoryRequestDto)
     Добавляет новую категорию. Проверяет корректность данных и уникальность имени категории.
     При успешном добавлении сохраняет категорию в БД и возвращает DTO-объект CategoryResponseDto.

   • updateCategory(Integer categoryId, CategoryRequestDto categoryRequestDto)
     Обновляет существующую категорию. Проверяет наличие категории по id и уникальность нового имени.
     Возвращает обновлённый объект CategoryResponseDto.

   • deleteCategory(Integer categoryId)
     Удаляет категорию по идентификатору. Перед удалением снимает связь с продуктами, у которых была эта категория.
     Возвращает DTO удалённой категории.

   • getAllCategories()
     Возвращает список всех категорий в виде списка DTO.

   • getCategoryById(Integer categoryId)
     Ищет категорию по идентификатору. Если категория не найдена — выбрасывает исключение BadRequestException.

   • getCategoryByName(String categoryName)
     Ищет категорию по имени. Если категория не найдена — выбрасывает исключение BadRequestException.

4. Используемые исключения:
   - IllegalArgumentException — если обязательные параметры не переданы или некорректны.
   - BadRequestException — если категория уже существует, не найдена или нарушена уникальность имени.

5. Особенности:
   - Методы addCategory, updateCategory и deleteCategory аннотированы @Transactional, что гарантирует целостность данных.
   - В методе deleteCategory присутствует TODO: заменить присвоение null для продуктов на категорию по умолчанию.
   - Класс использует Lombok-аннотацию @RequiredArgsConstructor для автоматического создания конструктора с final-полями.
   - Обозначен как @Service — компонент Spring, содержащий бизнес-логику.

6. Вывод:
   CategoryService является ключевым сервисом для работы с категориями товаров и обеспечивает безопасное управление их состоянием,
   взаимодействуя с уровнем данных (Repository) и DTO-объектами для передачи данных между слоями приложения.
